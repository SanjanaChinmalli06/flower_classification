How it Works

Understand the Iris prediction flow step by step

This page explains how data enters the system, how the models are trained, and why both predictions are shown for the same input.

1. Input measurement

You enter the sepal length, sepal width, petal length, and petal width of an Iris flower.

2. Pre-trained models

The app uses two models trained on the Iris dataset: Logistic Regression and Decision Tree.

3. Dual prediction

Both models predict the species independently, giving you a direct comparison for the same flower measurements.

Why two models?

Logistic Regression creates a smooth decision boundary by weighing all features together. It works well when the classes separate in a mostly linear way.

Decision Tree splits the data using yes/no rules on individual feature values. It is better at capturing non-linear patterns, but it can behave differently if the input sits close to a boundary.

Showing both results helps you compare model behavior and understand which species assignment is more consistent for that input.

4. Accuracy explained

The Results page shows each model's accuracy on a holdout test set, so you can see their relative performance.

5. Confusion matrices

Confusion matrices show where each model is strong and where it makes mistakes between Iris species.

Use the prediction page for immediate comparison, or visit the results page for deeper performance analysis.
